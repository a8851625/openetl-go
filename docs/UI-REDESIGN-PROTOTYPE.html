PIPELINE WORKSPACE · 非一级导航

高级 DAG 编辑

DAG 编排没有删除。它从「与仪表盘同级的产品入口」降级为 pipeline 的高级工作区。

为什么不再做一级「DAG 编排」页？ 产品主表面是 Source → Transform → Sink 管道；YAML / API / UI 操作同一份 spec。 DAG、调度、分片、master-worker 是该模型的扩展，不形成独立产品线。 一级导航放 Designer/DAG 会让新手先面对节点库，而不是「创建成功、稳定运行、快速修复」。 多源、条件路由、fanout 等能力仍在此画布完成，并写回同一 pipeline。
